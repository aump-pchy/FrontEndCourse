JavaScript พื้นฐาน

สอนเฉพาะ JS ที่ React ต้องใช้ใน Week นี้ — const/let, string, template literal

📦 const และ let — ประกาศตัวแปร
// let — เปลี่ยนค่าได้ let score = 0 score = 10 // ✅ เปลี่ยนได้ // const — เปลี่ยนค่าไม่ได้ const name = "สมชาย" name = "สมหญิง" // ❌ Error! // React ใช้ const เกือบทั้งหมด const title = "ProfileCard" const age = 20 const isStudent = true
const
ใช้บ่อยที่สุด — ค่าที่ไม่เปลี่ยนแปลง เช่น ชื่อ, ตัวเลข, function
let
ใช้เมื่อต้องการเปลี่ยนค่า เช่น คะแนน, สถานะ, counter
❌ ใน React ห้ามใช้ var — เป็น JS เก่ามีปัญหาเรื่อง scope ใช้แต่ const/let เท่านั้น
✏️ Template Literal — ใส่ตัวแปรใน string
const name = "สมชาย" const age = 20 const school = "วทล." // แบบเดิม — ต่อ string ด้วย + "สวัสดี " + name + " อายุ " + age + " ปี" // Template Literal — ใช้ backtick + ${} `สวัสดี ${name} อายุ ${age} ปี` `เรียนที่ ${school}` // ใส่ expression ใน ${} ได้เลย `ปีหน้าอายุ ${age + 1} ปี`
💡 ใช้ backtick ( ` ) ไม่ใช่ apostrophe ( ' ) — คีย์อยู่มุมบนซ้ายของคีย์บอร์ด
🔥 Template Literal ใช้ใน JSX บ่อยมาก เช่น className={`text-${color}-500`}
เทียบให้เห็น:
❌ "Hello " + name + "!"
✅ `Hello ${name}!`

➡️ Arrow Function — เขียน function แบบสั้น
// function แบบเดิม function greet(name) { return `สวัสดี ${name}` } // Arrow Function — เหมือนกันทุกอย่าง const greet = (name) => { return `สวัสดี ${name}` } // ถ้า return บรรทัดเดียว ย่อได้ const greet = (name) => `สวัสดี ${name}` // เรียกใช้เหมือนกันเลย greet("สมชาย") // → "สวัสดี สมชาย"
💡 React Component เขียนด้วย Arrow Function ทุกตัว — ต้องชินกับ syntax นี้
เทียบ syntax:
function เดิมfunction Fn() {}
arrow functionconst Fn = () => {}

React คืออะไร?

Library สำหรับสร้าง UI — สร้าง Component แล้วนำมาประกอบเป็นหน้าเว็บ

🤔 ทำไมต้องใช้ React?
เว็บปกติที่เขียนด้วย HTML ล้วน ถ้าอยากอัปเดตข้อมูลต้องโหลดหน้าใหม่ทุกครั้ง

React ทำให้ อัปเดตแค่ส่วนที่เปลี่ยน โดยไม่ต้อง reload — เร็วและลื่นกว่ามาก
⚛️ React — Library ของ Facebook/Meta (2013)
🏢 ใครใช้ — Facebook, Instagram, Airbnb, Netflix
💼 ตลาดงาน — เป็นทักษะที่นายจ้างหาบ่อยที่สุด
แนวคิด Component-based:
// หน้าเว็บ = ประกอบจาก Component
<App>
<Navbar />
<HeroSection />
<MenuGrid>
<MenuCard /> ← สร้างครั้งเดียว
<MenuCard /> ← ใช้ซ้ำได้เลย
<MenuCard />
</MenuGrid>
<Footer />
</App>
💡 Component = สร้างครั้งเดียว ใช้ซ้ำได้ทุกที่ — เหมือน Lego ประกอบหน้าเว็บ
⚖️ HTML ธรรมดา vs React
HTML ธรรมดาReact
เขียนซ้ำถ้าต้องการหลาย card สร้าง <MenuCard /> ครั้งเดียว เรียกใช้ซ้ำได้
อัปเดตข้อมูลต้อง JS + DOM manipulation React อัปเดต UI ให้อัตโนมัติเมื่อข้อมูลเปลี่ยน
ไฟล์ใหญ่ขึ้นเรื่อยๆ จัดการยาก แบ่งเป็น component เล็กๆ หาง่าย แก้ง่าย
ไม่มีระบบ state/data management มี useState/props จัดการข้อมูลได้ชัดเจน

JSX + Function Component

เขียน HTML ใน JavaScript — แล้วส่งข้อมูลเข้า Component ด้วย Props

🧬 JSX คืออะไร?
JSX = JavaScript + XML
เขียน HTML ลงไปใน JS ได้เลย — React จะแปลงให้เอง
// JS ปกติ — return string const text = "สวัสดี" // JSX — return HTML-like const element = ( <div className="card"> <h1>สวัสดี</h1> <p>ยินดีต้อนรับ</p> </div> )
ข้อแตกต่าง HTML vs JSX:
HTMLJSX
class="..."className="..."
for="..."htmlFor="..."
<br><br /> ต้องปิด
<img><img /> ต้องปิด
style="color:red"style={{"{{"}}color:'red'{{"}}"}}
⚠️ ใช้ className แทน class เสมอ — ถ้าลืมจะ error ทันที
🧩 Function Component — หน่วยพื้นฐานของ React
// src/components/ProfileCard.jsx const ProfileCard = () => { return ( <div className="card"> <span>👨‍💻</span> <h2>สมชาย</h2> <p>นักศึกษา ปวส.1</p> </div> ) } export default ProfileCard
// src/App.jsx — นำมาใช้ import ProfileCard from './components/ProfileCard' const App = () => { return ( <div> {/* ใช้เหมือน HTML tag */} <ProfileCard /> <ProfileCard /> <ProfileCard /> </div> ) } export default App
💡 Component ชื่อต้องขึ้นต้นตัวพิมพ์ใหญ่ — <ProfileCard /> ไม่ใช่ <profilecard />
🔀 {} ใส่ค่าใน JSX + Props ส่งข้อมูล
{} ในไม่ใส่ string ธรรมดา — ใส่ JS expression
const name = "สมชาย" const age = 20 return ( <div> {/* ตัวแปร */} <h2>{name}</h2> {/* คำนวณ */} <p>อายุ {age + 1} ปี</p> {/* template literal */} <p>{`สวัสดี ${name}!`}</p> {/* เงื่อนไข */} <p>{age >= 18 ? "ผู้ใหญ่" : "เยาวชน"}</p> </div> )
Props — ส่งข้อมูลเข้า Component
// ส่ง props เข้าไป <ProfileCard name="สมชาย" role="ปวส.1" emoji="👨‍💻" /> // รับ props ใน Component const ProfileCard = (props) => { return ( <div> <span>{props.emoji}</span> <h2>{props.name}</h2> <p>{props.role}</p> </div> ) }
