HTML Semantic Elements

เขียน HTML ให้มีความหมาย — ไม่ใช้แต่ <div>

🤔 ทำไมต้องใช้ Semantic Tag?
❌ แบบเดิม — div ทุกอย่าง
<!-- ไม่รู้ว่าแต่ละส่วนคืออะไร --> <div class="header">...</div> <div class="nav">...</div> <div class="main">...</div> <div class="footer">...</div>
✅ แบบ Semantic — บอกความหมาย
<!-- Browser, Google, Accessibility รู้ทันที --> <header>...</header> <nav>...</nav> <main>...</main> <footer>...</footer>
💡 Semantic HTML ทำให้ Google จัดอันดับเว็บได้ดีขึ้น, คนพิการที่ใช้ Screen Reader อ่านเว็บได้, และตัวเองอ่านโค้ดกลับมาเข้าใจได้ง่ายกว่า
🔖 Tags Semantic ที่ต้องรู้
Tagใช้สำหรับตัวอย่าง
<header>ส่วนหัวของหน้าหรือ section — โลโก้, ชื่อเว็บด้านบนสุดของหน้า
<nav>แถบ Navigation — ลิงก์เมนูต่างๆเมนูด้านบน, sidebar
<main>เนื้อหาหลักของหน้า — มีได้แค่ 1 ตัวต่อหน้าพื้นที่เนื้อหาหลัก
<section>แบ่งเนื้อหาเป็นหัวข้อ — ควรมี headingHero, About, Contact
<article>เนื้อหาที่ standalone ได้ — อ่านแยกออกมาก็ได้ความบทความ, Product Card
<aside>เนื้อหาข้างเคียง — เสริมเนื้อหาหลักSidebar, Related posts
<footer>ส่วนท้ายของหน้าหรือ sectionCopyright, ลิงก์ท้ายหน้า
⚠️ Semantic tag ไม่มีสไตล์พิเศษ — แสดงผลเหมือน div เลย แต่มีความหมายทาง HTML ที่ Browser และเครื่องมืออื่นๆ เข้าใจ
🗂 โครงสร้างหน้าเว็บ — มองเห็นภาพ
<header> โลโก้ + ชื่อเว็บ
<nav> เมนู Home | About | Contact
<main>
<section> Hero
<section> Cards
<aside> Sidebar
<footer> Copyright

Workshop 1 — โครงสร้าง Semantic
HTML Semantic

แปลง <div> ที่มีอยู่ให้เป็น Semantic tag ที่ถูกต้อง แล้วเขียนโครงสร้างพื้นฐานของหน้าเว็บ cafe.html จาก Week 2 ใหม่โดยใช้ Semantic

1
พื้นฐาน: เปิดไฟล์ cafe.html ใส่ semantic tag ให้ถูกต้อง — เปลี่ยน div.header → <header>, div.nav → <nav>, div.content → <main>, div.footer → <footer>
2
กลาง: แบ่งส่วนเมนูอาหารใน <main> ออกเป็น <section> ตามหมวดหมู่ (เครื่องดื่ม, ขนม, อาหาร) แต่ละ section มี <h2> เป็น heading
3
ท้าทาย: ใช้ <article> สำหรับแต่ละรายการเมนู ภายใน article มี <h3> ชื่อเมนู, <p> คำอธิบาย, <footer> แสดงราคา
02

CSS Flexbox

จัดตำแหน่ง Element ได้ง่ายด้วย Flexible Box Layout

🧠 Flexbox คืออะไร?

ก่อนมี Flexbox การจัดตำแหน่ง element บนเว็บทำได้ยากมาก

Flexbox คือระบบ layout ที่ทำให้ items ข้างในกล่องจัดเรียงตัวเองได้อัตโนมัติ ตามที่เราต้องการ — จัดกลาง, จัดชิดขอบ, เว้นระยะ — ทำได้หมดด้วย CSS ไม่กี่บรรทัด

/* เปิดใช้ Flexbox */ display: flex; /* แค่นี้ ลูกข้างในจะเรียง */ /* เป็นแถวนอนแล้ว ✓ */
💡 กล่องที่ใส่ display:flex เรียกว่า Flex Container — ของข้างในเรียกว่า Flex Items
↔️ แกน Main Axis และ Cross Axis

Flexbox มี แกน 2 แกน เสมอ — แกนหลักที่ items วิ่ง และแกนตั้งฉากกัน

flex-direction: row (ค่าเริ่มต้น)

จัดตำแหน่งบน Main Axis (แกนหลัก)
justify-content: flex-start; /* ชิดซ้าย */ justify-content: center; /* กลาง */ justify-content: flex-end; /* ชิดขวา */ justify-content: space-between; /* เว้นขอบ */ justify-content: space-evenly; /* เว้นเท่ากัน */
align-items
จัดตำแหน่งบน Cross Axis (แกนตั้งฉาก)
align-items: flex-start; /* ชิดบน */ align-items: center; /* กลาง */ align-items: flex-end; /* ชิดล่าง */ align-items: stretch; /* ยืดเต็ม (default) */

Workshop 2 — สร้าง Navbar ด้วย CSS Flex
CSS Flexbox

สร้างไฟล์ navbar.html ใหม่ แล้ว build Navbar โดยใช้ CSS Flexbox เขียน style ใน <style> tag ตรงๆ (ยังไม่ใช้ Tailwind)

1
พื้นฐาน: สร้าง Navbar ที่มี โลโก้ชื่อร้าน (ซ้าย) และลิงก์เมนู 3 อัน (ขวา) โดยใช้ display:flex + justify-content:space-between + align-items:center
2
กลาง: เพิ่ม Hero section ใต้ Navbar — ข้อความ + ปุ่ม จัดกึ่งกลาง ทั้ง horizontal และ vertical โดยใช้ flex-direction:column + align-items:center + justify-content:center
3
ท้าทาย: เพิ่ม Card section 3 ใบโดยใช้ flex + gap และใช้ flex:1 ให้ card ขยายเท่ากันหมด

Workshop 3 — แปลง CSS → Tailwind
Tailwind Flex

เปิด navbar.html จาก Workshop 2 แล้วแปลง CSS flex ทั้งหมดให้เป็น Tailwind class โดยไม่แก้ผลลัพธ์ที่เห็นบนหน้าจอ

1
พื้นฐาน: เพิ่ม Tailwind CDN แล้วแปลง display:flex → class="flex", justify-content:space-between → justify-between, align-items:center → items-center บน Navbar
2
กลาง: ลบ <style> tag ทั้งหมดออก แล้วใส่ Tailwind class ให้ครบจนหน้าตาเหมือนเดิม รวมถึง gap, padding, background สี
3
ท้าทาย: เพิ่ม hidden md:flex บน div ลิงก์ เพื่อซ่อนเมนูบนหน้าจอเล็ก และเพิ่มปุ่ม hamburger ที่ซ่อนบน md ขึ้นไป (md:hidden)
