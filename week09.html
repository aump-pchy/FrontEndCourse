JavaScript Object

เก็บข้อมูลหลายอย่างไว้ในตัวแปรเดียว — เหมาะกับข้อมูลที่เกี่ยวข้องกัน

🤔 ทำไมต้องมี Object?
❌ แบบเดิม — ตัวแปรกระจาย
// ข้อมูลกาแฟ 1 แก้ว ใช้ 4 ตัวแปร const name = "ลาเต้" const price = 80 const emoji = "☕" const isBestseller = true // ถ้ามี 10 เมนู = 40 ตัวแปร 😱
✅ ใช้ Object — มัดรวมกัน
// ข้อมูลทั้งหมดอยู่ใน object เดียว const latte = { name: "ลาเต้", price: 80, emoji: "☕", isBestseller: true } // อ่านค่า latte.name // → "ลาเต้" latte.price // → 80
💡 Object เหมาะเมื่อข้อมูลเป็น "ของชิ้นเดียวที่มีหลาย property" — เมนูอาหาร, ข้อมูลนักเรียน, สินค้า
🔑 Dot Notation — เข้าถึงค่าใน Object
const menu = { name: "อเมริกาโน่", price: 65, emoji: "☕", category: "เครื่องดื่ม", isBestseller: false } // อ่านค่า menu.name // "อเมริกาโน่" menu.price // 65 menu.emoji // "☕" // ใช้ใน template literal `${menu.emoji} ${menu.name} ฿${menu.price}` // → "☕ อเมริกาโน่ ฿65" // แก้ค่า (ถ้าใช้ let) let item = { price: 65 } item.price = 70 // เปลี่ยนได้
Object ซ้อน Object ได้ (Nested)
const cafe = { name: "Morning Glow", location: { city: "เลย", zip: "42000" }, rating: 4.9 } // เข้าถึง nested cafe.location.city // "เลย" cafe.rating // 4.9
⚠️ ชื่อ key ใน object ไม่มีลำดับ — ต่างจาก array ที่มี index 0,1,2

02

Destructuring

แกะค่าออกจาก Object มาใส่ตัวแปรเดี่ยวๆ — เขียนโค้ดสั้นลงมาก

📦 Destructuring คืออะไร?
❌ แบบเดิม — ยาว
const menu = { name: "ลาเต้", price: 80, emoji: "☕" } // ต้องพิมพ์ menu. ทุกครั้ง const name = menu.name const price = menu.price const emoji = menu.emoji console.log(name, price, emoji)
✅ Destructuring — สั้นกว่ามาก
const menu = { name: "ลาเต้", price: 80, emoji: "☕" } // แกะออกมาทีเดียว const { name, price, emoji } = menu // ใช้ได้เลยโดยไม่ต้อง menu. console.log(name, price, emoji) // "ลาเต้" 80 "☕"
💡 ชื่อตัวแปรต้อง ตรงกับชื่อ key ใน object — {"{"}name{"}"} จะดึง key ชื่อ name ออกมา
🔧 เทคนิคเพิ่มเติม — เปลี่ยนชื่อ & ค่า Default
เปลี่ยนชื่อตัวแปร (rename)
const menu = { name: "ลาเต้" } // key : ชื่อใหม่ const { name: menuName } = menu console.log(menuName) // "ลาเต้" console.log(name) // ❌ Error! name ไม่มีอยู่
ค่า Default (ถ้า key ไม่มีใน object)
const menu = { name: "ลาเต้", price: 80 // ไม่มี emoji } const { name, price, emoji = "☕" // ถ้าไม่มี ใช้ค่านี้ } = menu console.log(emoji) // "☕" (default)

👆 คลิก checkbox ของ key ที่ต้องการ — ดู destructuring syntax และตัวแปรที่ได้

Code ที่ได้

← เลือก key ทางซ้าย

⚡ Destructuring ใน Function Parameter — ใช้บ่อยมากใน React
แบบเดิม — รับ props ทั้งก้อน
// ต้องพิมพ์ props. ทุกที่ const MenuCard = (props) => { return ( <div> <span>{props.emoji}</span> <h3>{props.name}</h3> <p>฿{props.price}</p> </div> ) }
✅ Destructuring — อ่านง่ายกว่า
// destructure ตรงใน parameter const MenuCard = ({ emoji, name, price }) => { return ( <div> <span>{emoji}</span> <h3>{name}</h3> <p>฿{price}</p> </div> ) }
🔥 นี่คือ pattern ที่ React developer ใช้บ่อยที่สุด — destructure ตรงใน parameter เลย

Props ขั้นสูง

ส่ง Object เป็น Props และใช้ Spread Operator ให้โค้ดสั้นลงครึ่งนึง

🔀 3 วิธีส่ง Props — เทียบให้เห็น
❌ แบบที่ 1 — Hardcode
// App.jsx <MenuCard name="ลาเต้" price={80} emoji="☕" isBestseller={true} />
เปลี่ยนข้อมูลทีต้องแก้ไฟล์
✅ แบบที่ 2 — จาก Object
const latte = { name: "ลาเต้", price: 80, emoji: "☕", isBestseller: true } <MenuCard name={latte.name} price={latte.price} emoji={latte.emoji} />
ข้อมูลอยู่ที่เดียว แก้ง่าย
🔥 แบบที่ 3 — Spread Operator (ดีที่สุด)
latte = {"{"} name, price, emoji, isBestseller {"}"}
→
<MenuCard {"{"}...latte{"}"} />
const latte = { name:"ลาเต้", price:80, emoji:"☕", isBestseller:true } // ... คือ "กระจาย" ทุก key ออกมาเป็น props อัตโนมัติ <MenuCard {...latte} /> // เหมือนกับเขียนแบบนี้เลย: <MenuCard name={latte.name} price={latte.price} emoji={latte.emoji} isBestseller={latte.isBestseller} />
🔥 {"{"}...object{"}"} = Spread Operator — กระจาย key ทั้งหมดออกมาเป็น props ทีเดียวเลย ประหยัดการพิมพ์มาก
📋 Array of Objects — ข้อมูลหลายชิ้น
// App.jsx — เก็บข้อมูลเป็น array const menuData = [ { id: 1, name: "ลาเต้", price: 80, emoji: "☕", isBestseller: true }, { id: 2, name: "อเมริกาโน่", price: 65, emoji: "☕", isBestseller: false }, { id: 3, name: "บราวนี่", price: 65, emoji: "🍫", isBestseller: false } ] // ใช้ Spread ส่ง props <div className="grid grid-cols-3 gap-4"> <MenuCard {...menuData[0]} /> <MenuCard {...menuData[1]} /> <MenuCard {...menuData[2]} /> </div>
🔮 Week 11 จะสอนใช้ .map() render อัตโนมัติ — แทนที่จะเรียกทีละ [0], [1], [2]
MenuCard.jsx ที่รับ props:
const MenuCard = ({ name, price, emoji, isBestseller = false // default }) => { return ( <div className="bg-white rounded-xl shadow-md overflow-hidden"> {/* Badge ถ้าขายดี */} {isBestseller && ( <div className="text-xs text-green-600 border border-green-400 rounded-full px-2 py-0.5 inline-block m-2"> ⭐ ขายดี </div> )} <div className="text-4xl p-4">{emoji}</div> <div className="px-4 pb-4"> <h3 className="font-bold text-gray-800">{name}</h3> <p className="text-orange-500 font-semibold">฿{price}</p> </div> </div> ) } export default MenuCard

❓ Conditional Rendering — แสดงตามเงื่อนไข
// && — แสดงเมื่อเงื่อนไขเป็น true {isBestseller && ( <span>⭐ ขายดี</span> )} // ternary — true/false ต่างกัน {isBestseller ? <span>⭐ ขายดี</span> : <span>สินค้าทั่วไป</span> } // แสดง className ตามเงื่อนไข <p className={` font-bold ${isBestseller ? "text-green-600" : "text-gray-800"} `}> {name} </p>
💡 {"{"}condition && <JSX />{"}"} — แสดงเฉพาะเมื่อ condition เป็น true ใช้บ่อยมากสำหรับ badge, label, icon พิเศษ
